# 1. 决策树用来解决什么问题？

感知机用于**预测数据所属的类别**，可用于多分类问题。特别的，决策树的“决策过程”非常直观，可解释性很强。但与之相对的是容易过拟合，预测类别的效果一般。详细的特性及应用场景见小结部分。

# 2. 决策树的模型是什么？

决策树可以看出是一系列`if-elif-else`的判断集合。决策树的路径或其对应的`if-elif-else`规则集合具有 一 个重
要的性质 : **互斥并且完备** 。 我们用一段Python代码感受一下决策树的“预测逻辑”。以下是某个女生选择是否见相亲对象的逻辑：

```python
def 是否相亲(年龄, 长相, 收入, 工作) -> bool:
    if 年龄 > 30:
        return False
    else:
    	if 长相 == "丑":
        	return False
        else:
    		if 收入 == "低":
        		return False
    		elif 收入 == "高":
        		return True
    		else:
        		if 工作 == "公务员":
             		return True
        		else:
             		return False
```

在这里，年龄、长相、收入和工作实际上就是决策树的“**内部节点**”；最后得到的类别(这里恰好是二分类)：True和False就是决策树的“**叶子节点**”。而“判断”实际就是一个内部节点所生长出来的一条“**有向边**”。比如，“收入”这个内部节点下面就生长出了三条有向边。“内部节点”、“叶子节点”和“有向边”共同构成了一颗决策树模型。

需要补充，决策树模型中的“叶子节点”不一定是确定性的类别，也可以是**条件概率分布**$P(Y | X)$。比如当年龄大于30时，返回的内容可以是：$P(Y = "不见" | 年龄 >= 30) = 0.8$。而从另外一个角度看，确定性的类别可以看作是极端的条件概率分布，即年龄大于30时的类别判断“见”实际上等价于$P(Y = "不见" | 年龄 >= 30) = 1$。

# 3. 决策树的策略是什么？

从决策树的模型中叶子节点可能是条件概率分布就知道，一条确定性的决策路径可能是不存在的，因此才用条件概率去描述最终这个不确定的结果。比如，数据中可能存在不管收入高低，女生都去见相亲对象的情况。此时，收入底下的三条边都无法对应一个确定性的分类，只能是一个条件概率分布了。

又从决策树模型中同一个特征可以多次充当“内部节点”，如果让决策树不加节制的生长，那么就会让模型过拟合。

因此，决策树的损失函数需要同时兼顾这种“预测结果的确定性”和“决策树的简洁性”。损失函数如下：
$$
C_\alpha(T) = \sum_{t=1}^{|T|} N_t H_t(T) + \alpha |T|
$$

- 第一部分：对训练数据的拟合误差（经验风险）：$\sum_{t=1}^{|T|} N_t H_t(T)$

    - **$|T|$**：决策树的叶节点个数。
    - **$N_t$**：叶节点 $t$ 包含的样本个数。
    - **$H_t(T)$**：叶节点 $t$ 上的**经验熵**。它衡量了该节点内样本类别的“混乱程度”，也可以理解为预测类别的“确定性”。
    - 这个求和项计算的是**整个决策树的预测误差**。$N_t$ 作为权重，意味着样本越多的节点，其“混乱”对整体损失的影响越大。

- **第二部分：对模型复杂度的惩罚（结构风险）**：$\alpha |T|$
- **$\alpha \ge 0$**：**正则化系数**。它是一个超参数，用来控制惩罚的力度。
    - **$|T|$**：直接用叶节点的数量来度量模型的复杂度。叶节点越多，树越深，模型越复杂，这一项的惩罚就越大。

因此，决策树的策略是让决策树模型在经验风险和结构风险之间达到一个良好的平衡。

# 4. 决策树的算法是什么？

## 4.1 特征选择

由前面可知，决策树包括了“内部节点”和“叶子节点”。特征选择实际上就是在决定这些内部节点的先后顺序。比如前面相亲的例子，是先通过**年龄**划分，还是先通过**长相**划分呢？这取决于哪个特征对预测类别的“**区分度**”最好。

“区分度”的一个感性理解是：假设收入高的相亲对象女生都选择去见，收入一般或较低的相亲对象女生都不选择去见，那么“收入高低”就具有很强**区分度**。度量一个特征“区分度”，又包含了信息增益(ID3)、信息增益比(C4.5)两种方法。

## 4.2 树的生成

由于最优决策树的生成被证明是一个NP-Complete问题，因此在现实情况中我们无法真正求出最优的决策树。通常的做法是用“贪心算法”，即**每一次都选择在当前数据看来是最好的特征**，然后划分数据；在每一批划分的数据中继续选择最优特征。这样一直递归进行下去，直到达到某一终止条件。

例如，一开始我们得到“年龄”是当前所有数据中最具有区分度的特征，随后原本的数据根据是否年龄是否大于30岁分成了两批。因为年龄大于30的那批相亲对象，女生都不选择去见，因此直接结成了“叶子节点”；剩下我们需要继续在年龄小于等于30的那批数据中，看长相、收入和工作谁是对当前数据最有区分度的特征。

## 4.3 树的剪枝

一颗决策树越复杂，其对应的“经验风险”越小。换言之，它对训练数据的预测效果就越好。但与此同时，这颗决策树的泛化能力也会迅速衰减。这也是我们前面说的要平衡“经验风险”和“模型复杂度”的原因。

限制模型复杂度的方法有很多，直接约束如“限制树的最大深度”，间接约束如“规定内部节点再划分所需最少样本数”、“叶子节点最少样本数”等。但这些方法都只是相当于在树的生成时设立终止条件，也可以叫他们“**预剪枝**”。在树生成完成后，我们可以选择继续简化决策树，比如减掉一些对决策影响比较小的叶子节点，然后将这个叶子节点的上一级设为叶子节点。这一步也被叫作“**后剪枝**”。

在工业界中，“后剪枝”的效果会好一些，但由于其计算复杂，通常不予考虑。“预剪枝”是我们更为常用的手段。

## 4.4 步骤总结

1. 构建根结点， 将所有训练、数据都放在根结点。**选择一个最优特征**，按照这一特征将训练数据集分割成子集 ， 使得各个子集有一个在当前条件下最好的分类。
2. 如果这些子集已经能够被基本正确分类，那么构建叶结点，并将这些子集分到所对应的叶结点中去。如果还有子集不能被基本正确分类，那么就对这些子集选择新的最优特征，继续对其迸行分割， 构建相应的结点。
3. 如此**递归地**进行下去，直至所有训练数据子集被基本正确分类，或者没有合适的特征为止。
4. 对已生成的树白下而上进行**剪枝**，将树变得更简单，从而使它具有更好的泛化能力。具体地，就是去掉过于细分的叶结点，使其回退到父结点，甚至更高的结点，然后将父结点或更高的结点改为新的叶结点。  

# 5. 概念补充

## 5.1 信息量

假如系统中有的$N$个等可能事件，那么至少需要 $\mathrm{log}_2N$个信号，才能传递该事件发生的信息，我们可以称之为该事件的**信息量**。由于等可能事件发生的概率为$\frac{1}{N}$，因此信息量的公式可以写作$\mathrm{log}_2 1 / (1 / N)$。推广可知：假设时间$x$的发生概率为$p(x)$，则传递该事件发生需要的最少比特信号位（即信息量）为：

$$
\mathrm{log}_2 (1 / p(x)) = -\mathrm{log}_2p(x)
$$

## 5.2 熵

**信息熵用于衡量整个事件空间包含的平均信息量，即信息量的数学期望。**例如随机变量$X$的熵是：
$$
H(x) = E(-\mathrm{log}_2 p(x)) = -\sum_{i=1}^n p_i \cdot \mathrm{log}_2 p(x)
$$

## 5.3 条件熵

已知分类变量$X$和$Y$之间可能存在相关关系。那么你在得知$X$的取值的情况下，$Y$的取值平均对你而言还有多“意外”呢？我们可以设想三种情形。注意，由于分类变量没有对应的相关系数作为相关性的度量，我们临时套用数值型变量的相关系数符号$\rho$，大家意会即可：

1. 当$\rho_{xy} = 0$，随机变量$X$的值对我们知道$Y$的值没有任何帮助，因此$Y$的取值平均对我而言还是和原来一样；
2. 当$\rho_{xy} \in (0, 1)$时，随机变量$X$的取值在一定程度上影响$Y$的取值，我可以借助$X$预测$Y$，因此$Y$的取值平均而言对我的“意外”程度会变小一些；
3. 当$\rho_{xy} = 1$时，随机变量$X$的取值直接确定$Y$的取值。也就是说，知道$X$以后，$Y$的取值不会给我任何意外，即“意外程度”为0；

上述正是条件熵衡量的基本内容与性质，表明了$H(Y | X) \leq H(Y)$的；它的严格定义如下：
$$
H(Y | X) = \sum_{x \in X} p(x) H(Y | X = x)
$$

```python
def H(df: pd.DataFrame, Y: str, X: str = "") -> np.float64:
    if X == "":
        # 计算熵
        return -(df[Y].value_counts(normalize=True)).apply(lambda x: x * math.log2(x)).sum()
    else:
        # 计算条件熵
        cond_df = df[X].value_counts(normalize=True).reset_index()
        return (cond_df["proportion"] * cond_df[X].apply(lambda x: H(df[df[X] == x], Y))).sum()
```

## 5.4 信息增益

既然引入一个外部条件$X$，$Y$的取值给我们带来的意外程度会下降。这部分下降的“意外程度”可以看出是$X$给我们带来的信息，也就是信息增益，定义如下：
$$
g(Y, X) = H(Y) - H(Y | X)
$$

```python
def g(df: pd.DataFrame, Y: str, X: str) -> np.float64:
    return H(df, Y) - H(df, Y, X)
```



## 5.5 信息增益比

信息增益有一个致命缺陷：如果外部条件$X$的取值非常多，$g(Y, X)$必然会增大。一种极端情形，假设$X$是一个唯一id值，那么每一个$X$都能对应一个$Y$。数据上看$X$和$Y$构成了强相关关系，但这实际上却没有任何意义。

因此我们需要乘以一个项$\alpha$去惩罚信息增益，构造一个新的统计量$g_R(Y, X)$。要求是：外部条件$X$的取值情况增多时，$\alpha$要变小。此外，我们希望$\alpha \in [0, 1]$，以避免量纲发生变化。而$\frac{1}{H(X)}$正完美符合这个要求！

所以，信息增益比实际上就是：
$$
g_R(Y, X) = \frac{g(Y, X)}{H(X)} = \frac{H(Y) - H(Y |X)}{H(X)}
$$

```python
def g_R(df: pd.DataFrame, Y: str, X: str) -> np.float64:
    return g(df, Y, X) / H(df, X)
```



## 5.6 基尼系数/基尼不纯度

为了解决熵计算复杂的问题（计算对数比较慢），工业界更倾向于使用基尼不纯度。基尼不纯度不仅计算极快，而且是工业标准算法，如它是`scikit-learn`库中决策树(CART)算法的默认指标。

基尼不纯度可以简单理解为随机从当前数据中地有放回地抽取两个样本，两个样本的某个特征取值不同的概率的**期望**。例如，前面相亲的例子中，收入有高、中、低三种情况。假设收入高的有3个人，收入中等的有8个人，收入低的有5个人。记“收入”为$X$，那么，抽到两个样本，它们的收入取值不同的概率就是：
$$
\begin{aligned}
p &= P(X=高)P(X=中 | X=低) + P(X=中)P(X=高 | X=低) + P(X=低) P(X=高 | X = 中) \\
&= \frac{3}{16} \times \frac{13}{16} + \frac{8}{16} \times \frac{8}{16} + \frac{5}{16} \times \frac{10}{16} = \frac{153}{256}
\end{aligned}
$$

对上述过程作出抽象。记特征$X$有$k$个取值，每个取值的概率是$p_k$，基尼指数的定义就是：
$$
\mathrm{Gini}(X) = \sum_{i=1}^k p_k (1 - p_k) = 1 - \sum_{i=1}^k p_k^2
$$



# 6. 小结

## 6.1 核心工程特性

| 特性           | 说明                                                         |
| -------------- | ------------------------------------------------------------ |
| **输入要求**   | 数值型 + 类别型（部分实现需编码），**无需归一化/标准化**等预处理 |
| **缺失值处理** | 原生支持（如 C4.5、sklearn 可走代理分支）                    |
| **训练速度**   | O(n * m * log m) 级别，百万级以内数据可接受                  |
| **预测速度**   | O(depth)，非常快，内存友好                                   |
| **可解释性**   | 强，可导出 IF‑THEN 规则，支持可视化                          |
| **过拟合风险** | 高，必须配合剪枝或深度限制                                   |
| **稳定性**     | 差，数据微小变动可能改变树结构                               |

## 6.2 何时适合用？

1. **业务方要求“白盒模型”** 
   金融信贷、医疗诊断、反欺诈等合规或解释性场景。决策树可以直接输出决策路径：`“年龄>30 且 收入<5k → 拒绝”`，业务人员能理解并信任。
2. **数据预处理成本受限** 
   特征混合了数值（温度、金额）和类别（城市、等级），且没有精力做标准化、异常值处理。决策树对尺度不敏感，类别特征处理好（如 target encoding 或直接 One‑Hot）即可。
3. **快速原型验证** 
   数据量在几万～几十万，特征几十～几百维。决策树训练快，能快速给出基线模型和特征重要性排序。
4. **需要捕捉非线性、分段规律** 
   例如根据“年龄+收入”划分不同消费偏好，决策树天然适合分段常数型的决策边界。

## 6.3 何时不适合用？

| 场景                                       | 替代方案                                    |
| ------------------------------------------ | ------------------------------------------- |
| 高维稀疏数据（如文本、推荐）               | 线性模型、XGBoost（sparse aware）           |
| 数据量极大（> 500 万，特征 > 5000）        | XGBoost / LightGBM / 随机森林（单棵树太慢） |
| 要求预测概率平滑或外推                     | 逻辑回归、神经网络                          |
| 要求模型极端稳定（避免小扰动导致输出突变） | 集成模型或线性模型                          |
| 特征间存在复杂交互且需要斜向边界           | SVM 或神经网络                              |

## 6.4 核心参数

| 参数                | 作用                              | 推荐起始值                                       |
| ------------------- | --------------------------------- | ------------------------------------------------ |
| `max_depth`         | 限制树的最大深度                  | 分类：5～10；回归：10～20（视数据量）            |
| `min_samples_split` | 内部节点再划分所需最少样本数      | 2～10（数据大时取大）                            |
| `min_samples_leaf`  | 叶子节点最少样本数                | 1～5（防止过拟合）                               |
| `max_features`      | 每次划分考虑的特征数              | `sqrt(n_features)` 或 `log2`，提高随机性可降方差 |
| `ccp_alpha`         | 后剪枝复杂度参数（sklearn 0.22+） | 用 `cost_complexity_pruning_path` 找到合适的 α   |

## 6.5 未来展望

决策树直观，易解释，但单颗树稳定性差，容易过拟合。既然一棵树容易犯错，那么我们就种一种片森林。这也就是后面的“集成学习”的内容：

1. 随机森林(Random Forest)：构建很多棵相互独立的决策树，每棵决策树都是只抽取部分数据和特征构建的。面对未知数据时，所有数据独立预测，使用多数表决原则进行预测，既解决了决策树不稳定的问题，而且降低了误判风险。
2. 梯度提升树家族(GBDT / XGBoost / LightGBM)：树与树之间不再独立，而是前仆后继。第$K$棵树专门纠正第$(k - 1)$棵树的错误，不断迭代，将模型精度推向极致。
