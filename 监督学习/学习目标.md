## 一、多项式朴素贝叶斯 vs. 伯努利朴素贝叶斯

两者都是朴素贝叶斯在**离散特征**上的特例，区别在于**特征的取值类型**和**概率计算方式**。

| 对比维度               | 多项式朴素贝叶斯 (Multinomial NB)                            | 伯努利朴素贝叶斯 (Bernoulli NB)                              |
| ---------------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **特征取值**           | 非负整数，表示**频次**（例如词在文档中出现的次数）           | 二元值 {0,1}，表示**是否出现**                               |
| **数据分布假设**       | 特征向量服从**多项分布**（一个文档由词频向量生成）           | 每个特征服从**伯努利分布**（独立二元变量）                   |
| **条件概率估计**       | $P(x_i \mid y) = \frac{N_{yi} + \alpha}{N_y + \alpha \cdot V}$ <br>（$N_{yi}$：类$y$中特征$i$的总频次；$N_y$：类$y$中所有特征频次之和；$V$：特征总数；$\alpha$：拉普拉斯平滑参数） | $P(x_i=1 \mid y) = \frac{N_{yi}^+ + \alpha}{N_y + 2\alpha}$ <br>（$N_{yi}^+$：类$y$中包含特征$i$的文档数；$N_y$：类$y$的文档总数） |
| **未出现特征的处理**   | 频次为0，平滑后仍给一个小的概率                              | 同时建模出现（1）和不出现（0）的概率，即 $P(x_i=0 \mid y) = 1 - P(x_i=1 \mid y)$ |
| **对文档长度的敏感度** | 较敏感（长文档词频累计大，可能主导预测）                     | 不敏感（只关心是否出现）                                     |
| **典型应用场景**       | 文本分类中**词频**信息重要时（如新闻分类、主题识别）         | 短文本或**关键词存在与否**更重要时（如垃圾邮件检测、情感词极性） |

### 一个直观的例子

假设有两句话：
- 文档A：`"好吃 好吃 好吃"`（重复3次“好吃”）
- 文档B：`"好吃 难吃"`

**多项式NB**：会认为文档A中“好吃”的频次更高，更可能被分到“正面”类（如果“好吃”是正面词）。
**伯努利NB**：两篇文档都只包含“好吃”和“难吃”，但“好吃”在两篇中都出现了，所以伯努利无法区分强度，会认为它们相似。

## 二、如何应用于 NLP 文本分类

以**垃圾邮件分类**为例，步骤清晰如下：

### 1. 数据预处理
- 分词（中文可用 jieba，英文按空格分割）
- 转为小写、去除标点数字、停用词（如“的”、“是”、“the”）
- 可选：词干提取（英文）或保留原词

### 2. 特征提取（向量化）
- **多项式NB**：使用 **词频向量**（`CountVectorizer`）
  - 每个文档表示为一个长度等于词表大小的向量，每个位置是词在该文档中出现的次数。
  - 例：`{"好吃":3, "难吃":1}` → `[3, 1]`
- **伯努利NB**：使用 **二元出现向量**（`Binarizer` 或 `CountVectorizer(binary=True)`）
  - 只记录词是否出现（0或1）。
  - 例：`{"好吃":1, "难吃":1}`

### 3. 训练模型（沿用您文章中的拉普拉斯平滑）
- 计算**先验概率** $P(Y=c_k) = \frac{D_{c_k} + \alpha}{D + K \cdot \alpha}$（$D$总文档数，$K$类别数，$\alpha$常取1）
- 计算**条件概率**（平滑后）：
  - **多项式**：$P(w_i \mid c_k) = \frac{\text{count}(w_i, c_k) + \alpha}{\sum_{w \in V} \text{count}(w, c_k) + \alpha \cdot |V|}$
  - **伯努利**：$P(w_i=1 \mid c_k) = \frac{\text{doc\_count}(w_i, c_k) + \alpha}{D_{c_k} + 2\alpha}$  
    其中 $\text{doc\_count}(w_i, c_k)$ 是类别 $c_k$ 中包含词 $w_i$ 的文档数。

### 4. 预测新文档
- 对新文档做相同的分词、向量化（保持词表一致）。
- 计算每个类别的后验概率（取对数防止下溢）：
  - **多项式**：$\log P(c_k) + \sum_{i} x_i \cdot \log P(w_i \mid c_k)$  
    （$x_i$ 是词频）
  - **伯努利**：$\log P(c_k) + \sum_{i} \left[ x_i \log P(w_i=1 \mid c_k) + (1-x_i) \log P(w_i=0 \mid c_k) \right]$  
    （$x_i \in \{0,1\}$）
- 选择后验概率最大的类别。

### 5. 实际注意事项
- **平滑**：务必使用拉普拉斯平滑（$\alpha=1$），否则未登录词会导致概率为0。
- **数值稳定性**：使用对数概率，避免连乘下溢。
- **特征选择**：可去掉低频词（如出现次数<3）以减小词表，提高泛化。
- **文档长度归一化**（多项式）：有时用 TF-IDF 代替原始频次，可以减弱长度影响。

---

## 三、总结（延续文章风格）

> 朴素贝叶斯的“朴素”体现在特征条件独立假设上，而“多项式”和“伯努利”则是在此基础上对特征**分布形式**的不同约定。  
> - 如果你关心**词出现多少次**（比如长文档、主题模型），用**多项式NB**。  
> - 如果你只关心**词有没有出现**（比如短文本、关键词检测），用**伯努利NB**。  
> 两者都能结合拉普拉斯平滑解决零概率问题，且计算效率高，非常适合作为文本分类的**基线模型**。
