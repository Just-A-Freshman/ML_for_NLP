# 1. K近邻法用于解决什么问题？

K近邻法又称K-NN(K-Nearst-neighbor)，旨在**预测数据所属的类别**。和感知机的二分类不同，K近邻法可以进行**多分类**。

# 2. K近邻法的基本思想是什么？

K近邻法是一种监督学习的方法，需要一批已经分好类的数据集。其核心思想是，输入一个待预测的数据点，随后计算该数据点周围**最近的K个点**。随后判断这K个点中**大多数点属于哪个类别**，那么输入点就属于哪个类别。

# 3. K近邻法的模型是什么？

## 3.1 内容角度

从K近邻法的基本思想可以看出，该方法并没有预先构建一个带有参数的模型，没有需要更新的参数，因此就没有一个显式的学习过程。那么K近邻的模型是什么呢？

从模型分类来看，这就很自然引出了“参数模型”和“非参数模型”的区别。像那些带有参数的线性回归模型，就是我们接触得最多的“参数模型”。K近邻法显然不属于这类，它是典型的“非参数模型”。你可以认为，数据本身就是它的模型。当然也没这么简单，基本思想部分指出了几个关键点：

1. 距离度量：`最近的K个点`，“近”是需要用距离进行度量的。距离包括像$L_p$距离、名可夫斯基距离等；
2. 超参数K：预测是直接根据周围最近的K个点进行的，因此参数K是直接影响预测效果的。如何选取合适的超参数K，是这个模型的关键；
3. 分类决策规则：基本思想中的分类决策规则是`多数表决`，K个点大多数属于哪个类，预测类别就是哪个类；但这实际上是不唯一的，只是`多数表决`可以让0-1损失函数下的**经验风险最小**。

因此，总结来说，K近邻法的模型是`数据本身+距离度量+超参数K+分类决策规则`；

## 3.2 几何角度

让我们将K近邻法的模型想象成一张巨大的K维地图。根据模型，地图中的任意一个点都可以根据其周围标注数据确定其类别，这实际上构成了一个决策空间。我们来看一种简单的情形：

# 4. 怎么选择K值

1. 选择的K太小：此时只要待预测点周围有少许噪声点，预测结果就会有明显偏差，即估计误差变大，容易过拟合。
2. 选择的K值很大：优点是不易受噪声干扰了，估计误差变小。缺点是离预测点很远的点也会对结果构成影响，样本点本身的位置信息利用率低，更依赖多数决策规则。结果就是模型过于简单，近似误差变大。
3. 选择的K值适中：此时效果最佳。通常我们希望K值小一点，即优先保证模型具有一定的复杂度。随后使用**交叉验证法**从一个较小的K值开始往上确定K值。

# 5. 怎么实现K近邻法？

在确定距离度量的方法、超参数K以及分类决策规则后，预测本身就变成了一个数据**查询**问题：`怎么找到距离待预测的样本点最近的K个点`？

直接暴力搜索显然是不合适的，除非样本量很少，否则$O(n)$的计算复杂度根本吃不消。因此解法是寻找一个合适的数据存储结构。`KD`树便是书中所提到的一种实现方法。注意这里的K和`K近邻法`中的K是不同的。`KD`是指`K-Dimension`，即输入的特征向量有K个维度。

## 5.1. KD树的构造

先说一个感觉：KD树的构造过程更像是在为精确搜索而生的。

设想一个排序好的一维数组：$\{1, 3, 5, 6, 8, 9, 10\}$，如果要从中快速找到元素8，你会怎么找？一个很显然的想法是二分法。我们很容易在$O(logn)$的时间复杂度中找到8；但如果想知道元素7.5离谁更近呢？还是按照这套思路，我们很快就能锁定元素7.5在6和8之间。再稍微计算一下7.5和两个数之间的距离，就能知道8是最近的。所以，这套精确搜索的打法似乎可以迁移到距离搜索上来，它能很快找到一个和目标点比较接近的点。

现在的问题是：

1. 输入数据是K维的，而不是1维的。这意味着我们不能用排序好的一维数组存储数据。
2. 如果只对数据K个维度中的第W个维度进行排序，那么根据二分搜索，我们只能找到和目标点W维比较接近的点。但“距离近”的充分条件是各个维度都接近，只让某一个维度接近并不能保证距离是相近的。

KD树用“二叉树+逐维局部有序”来解决上述两个问题。二叉树的特点是“左小右大，节点二分叉”。我们先讲怎么做，再讲为什么。例子：
$$
\mathrm{samples} = {(7, 2), (5, 4), (2, 3), (4, 7), (9, 6), (8, 1)}
$$
记样本点$x_i = (x^{(1)}, x^{(2)})$，我们先排序$x^{(1)}$：
$$
\mathrm{sorted}(x_1^{(1)}, x_2^{(1)}, ..., x_n^{(1)}) = \mathrm{sorted(7, 5, 2, 4, 9, 8)} = (2, 4, 5, 7, 8, 9)
$$

1. 找到$x^{(1)}$的中间数，可以是5或7。假设我们固定选大的一方即7。那么根节点就是$(7, 2)$；然后，我们就将样本点一刀切成两份。左边是$x^{(1)} < 7$，右边是$x^{(1)} > 7$；所以现在根节点的左边$\{(2, 3), (4, 7), (5, 4)\}$，右边是$\{(8, 1), (9, 6)\}$。
2. 分别找到$(7, 2)$左侧和右侧$x^{(2)}$的排序后的中间数。左边是$\{3, 7, 4\}$，中间数是4；右边同理可得6；于是左边节点是$(5, 4)$，右边节点是$(9, 6)$；
3. 继续按照$x^{(2)}$ 划分。$(5, 4)$左边就是$(2, 3)$，右边就是$(4, 7)$；$(9, 6)$左边是$(8, 1)$，右边为空。

最后得到的树状图：

注意看这个存储的数据结构，每一个节点都存储了完整的向量数据，每一层的数据的一个维度都是有序的，并且左右相对平衡（得益于中间数的选取）。但它只是局部有序，如果我们把它完全展平成线性结构，比如严格地从左往右读$x^{(1)}$，会得到$\{2, 5, 4, 7, 8, 9\}$；这显然没有完全排序，但却非常接近有序了。

## 5. 2.  KD树的搜索

接下来看如何搜索。先说精确搜索。比如查$(4, 7)$。那么根据构造时的顺序，先看$x^{(1)}$；比7小，因此走左子树；再看$x^{(2)}$，比4大，因此走右子树，于是很快就找到了$(4, 7)$所在的位置。

再看最近邻搜索。比如查$(8, 3)$的最近邻：

1. 根据精确搜索的思路，快速下降到叶子节点$(8, 1)$。计算欧式距离的平方得到4，记为当前的最短距离；
2. 回溯到$(9, 6)$，计算距离为10。判断是否需要“下降”。由于$(9, 6)$的右子节点的$x^{(2)} >= 6$，而$(6 - 3)^2 = 9 > 4$，因此右侧节点距离不会更小，因此“剪枝”，无需下降；
3. 回溯到$(7, 2)$，计算距离为2 < 4，因此更新最短距离为2；
4. 由于$(7, 2)$左子节点的$x^{(1)} <= 7$，而$(8 - 7)^2 = 1 < 2$，因此必须下降检查；
5. 下降到$(5, 4)$，计算距离为10；简单判断可知无法剪枝，因此需要下降检查。因为$3 < 4$，所以先查左子树，计算距离为36；检查右子树距离为41；

因此，最后得到最近邻是$(7, 2)$；由于数据量较少，此次搜索仅仅通过剪枝减少了一次距离计算，接近于全量搜索。

## 5.3. KD树的思考

从思想来看，KD树用精确搜索的思路先找到一个和目标点在各维度上尽可能接近的点，从而快速确定一个较小的“最临近距离”，保证剪枝的效率。其搜索效率高度依赖于首次找到的这个“最临近点”以及剪枝条件满足情况。

先看最开头找到的这个“最临近点”。它是在每个维度上通过逐层二分来找到的。如果维度少，数据多，这么近似可谓相当有效。但是，试想一下，维度高一些呢？比如，63维。假设二叉树是平衡的，那么每下降一层能存储的数据就是上一层的两倍。这意味着除非我们的数据量达到的级别，否则连每一维进行二分逼近都无法做到。

再看剪枝。由于剪枝时只能查看其中一个维度的差距(点到超平面的距离)是不是已经比“最临近距离”都大了。这是一个很严苛的充分条件，在高维度几乎就无法满足。

因此，KD树只合适小维度，大数据量的简单场景。其依然有非常大的优化空间。

此外，5.1提到的KD树的构造过程是每个维度交替二分。为什么非得**每个维度**都去二分逼近呢？这实际上只有当数据足够均匀时才有合理性。因为这样每个维度都可以用二分逼近一下，最后得到的是最临近点的概率也更高。但是，倘若数据有哪怕一个维度上不是均匀的呢？试想一种极端情形，二维输入数据中，其中一个维度的数据的值全部是一个相同的值。换句话说，其中一个维度完全没有区分度。此时对它进行二分逼近是没有任何意义的。因此更自然的做法是按方差的大小进行切割，准确率也应该会更高。
